React(Hooks)+Typescript 类型定义实践

Tue Aug 31 2021
  • React
  • Typescript

React 结合 Typescript 的开发模式，在大型项目中运用比较广泛。本文借鉴（抄）了一些文章，就类型定义做了一些总结，有更多的可以补充。

# 数据类型定义

  • 基础类型
interface IProps {
  name: string; //字符串
  done: boolean; //布尔值
  age: number; //数字
  list: number[]; // 数组，定义元素类型
  optional?: string; // 可选参数
}
  • 对象类型
interface IProps {
  data: object; // 不推荐这么写，对象属性太模糊
  data1: {
    id: number,
    name: string
  }; // 对象属性类型定义，常用
  dataArr: {
    id: number,
    name: string
  }[]; // 对象数组常用
  dic: {
    [key: string]: customType
  }; // key值可以是任意字符串，值限制为customType自定义类型
  dic1: Record<string, customType>; // 和dic一样，用了TS内置的Record类型
}

其中 dic 和 dic1 的定义方式可以用于 ajax 请求的返回值，因为字段不确定，但 key 一般都是字符串类型

  • 函数类型
interface FunctionProps {
  /** 任意的函数类型 ❌ 不推荐 不能规定参数以及返回值类型 */
  onSomething: Function;
  /** 没有参数的函数 不需要返回值 😁 常用 */
  onClick: () => void;
  /** 带函数的参数 😁 非常常用 */
  onChange: (id: number) => void;
  /** 另一种函数语法 参数是 React 的按钮事件 😁 非常常用 */
  onClick(event: React.MouseEvent<HTMLButtonElement>): void;
}
  • React 内置类型

常用的内置类型：

interface IReact{
  style:React.CSSProperties //组件内联的style对象的类型
  node:React.ReactNode // 类似<div></div>的元素类型
  ele:React.React.ReactElement // 使用React.createElement创建的，可以简单理解为React中的JSX的元素
}

# 如何定义函数组件的类型

React 中有多种类型可以用来定义组件类型，接下来就结合实例来讲解各自的特点。

现在有一个 Hello 组件，接收一个 name 属性

// Hello.tsx
import React from 'react';

interface IProps {
  name: string;
}

const Hello = (props: IProps) => {
  return (
    <div>
      <div>hello!{props.name}</div>
    </div>
  );
};

export default Hello;

这里类型定义用interface或者type都可以，看个人喜好。如果用type，则这样写:

type IProps = {
  name: string;
};

然后，在父组件中渲染，并传递参数

// index.ts
import React from 'react';
import { render } from 'react-dom';
import Hello from './Hello';
import './style.css';

const App = () => {
  return (
    <div>
      <Hello name={'world'} />
    </div>
  );
};

render(<App />, document.getElementById('root'));

这样的情况下，类型的定义是没有问题的。

但现在有一个新的需求，需要在Hello组件中渲染其他新的组件。比如：

const App = () => {
  return (
    <div>
      <Hello name={'world'}>
        <button>click</button>
      </Hello>
    </div>
  );
};

这种情况，我们只需要在Hello组件中渲染children即可。

interface IProps {
  name: string;
  children:?//怎么去定义这个children类型呢
}

const Hello = (props: IProps) => {
  return (
    <div>
      <div>hello!{props.name}</div>
      {props.children}
    </div>
  );
};

chilren怎么定义类型？

  1. JSX.Element: 不推荐，单个子组件可用，没有考虑数组

  2. JSX.Element | JSX.Element[]: 不推荐，没有考虑字符串 children

  3. React.ReactChild[]: 没有考虑 null

  4. React.ReactNode：包含所有 children 情况

所以，这里children定义为React.ReactNode最为合适

interface IProps {
  name: string;
  children: React.ReactNode;
}

其实，对于函数组件而言，React 中还有一个 FC 内置类型，专门用于函数组件类型定义。用 FC 组件的好处是，它会自动包含 children ,因此我们不用手动去为 children 定义类型

import React, { FC } from 'react';

interface IProps {
  name: string;
}

// props参数解构，可以设置默认值
const Hello: FC<IProps> = ({ name = 'world', children }) => {
  return (
    <div>
      <div>hello!{name}</div>
      {children}
    </div>
  );
};

export default Hello;

综上，FC类型是函数组件类型定义的最佳选择。

# Hooks

  • useState

如果你的默认值已经可以说明类型，则不需要手动声明类型，TS 会自行推断。

// count: number类型
const [count, setCount] = useState(0);
const change = () => {
  setCount(count + 1);
};

有时候，我们需要设定初始值为null或undefined，此时就要通过范型手动传入你期望的类型。

import React, { useState } from 'react';

interface Person {
  name: string;
  age: number;
}

const [person, setPerson] = useState<Person | null>(null);

const getName = () => {
  console.log(person.name);
};

这样当你访问person上的属性时，会收到Object is possibly 'null'.的提示。通过?.可选链操作符语法可以避免这个错误。关于可选链操作法可以点击此处 (opens new window)查看

  • useEffect

useEffect传入的函数，它的返回值要么是一个方法（清理函数）,要么就是undefined，其他情况都会报错。

比较特殊的情况就是，useEffect中执行了一个async函数，比如:

// 提示Argument of type '() => Promise<void>' is not assignable to parameter of type 'EffectCallback'.
useEffect(async () => {
  const num = await getCount();
  setCount(num);
}, []);

因为 async 函数默认会返回一个 Promise,导致 TS 报错

推荐这样写：

useEffect(() => {
  const fn = async () => {
    const num = await getCount();
    setCount(num);
  };

  fn();
}, []);
  • useReducer

useReducer接收两个参数reducer 和initialState，只要对传入useReducer的reducer函数的state和action进行类型约束，就可以推断出类型。在使用useReducer的时候，联合类型就非常有用。

import React, { useReducer } from 'react';

const initialState = { count: 0 };

/** 联合类型，使用 | 连接 */
type ACTIONTYPE =
  | { type: 'increment'; payload: number }
  | { type: 'decrement'; payload: string }
  | { type: 'reset'; payload: number };

function reducer(state: typeof initialState, action: ACTIONTYPE) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + action.payload };
    case 'decrement':
      return { count: state.count - Number(action.payload) };
    case 'reset':
      return { count: action.payload };
    default:
      throw new Error();
  }
}

const Counter: React.FC = () => {
  const [state, dispatch] = useReducer(reducer, initialState);
  return (
    <>
      Count: {state.count}
      <button onClick={() => dispatch({ type: 'decrement', payload: '5' })}>-</button>
      <button onClick={() => dispatch({ type: 'increment', payload: 5 })}>+</button>
      <button onClick={() => dispatch({ type: 'reset', payload: 0 })}>reset</button>
    </>
  );
};

export default Counter;

这里，ACTIONTYPE是一个联合类型，当传入特定的type时，剩下的类型payload会自行匹配判断。

  • useRef

useRef 传递非空初始值的时候可以推断类型，可以通过传入第一个泛型参数来定义类型，约束 ref.current 的类型。

import React, { useLayoutEffect } from 'react';

export const App: React.FC = () => {
  const h1Ref = React.useRef<HTMLDivElement | null>(null);

  function changeInnerText(el: HTMLDivElement, value: string) {
    el.innerText = value;
  }

  useLayoutEffect(() => {
    if (null !== h1Ref.current) {
      changeInnerText(h1Ref.current, 'hello world');
    }
  }, []);

  return <h1 ref={h1Ref}>My title</h1>;
};

export default App;

更多元素类型实例：

// <div> reference type
const divRef = React.useRef<HTMLDivElement | null>(null);

// <button> reference type
const buttonRef = React.useRef<HTMLButtonElement | null>(null);

// <br /> reference type
const brRef = React.useRef<HTMLBRElement | null>(null);

// <a> reference type
const linkRef = React.useRef<HTMLLinkElement | null>(null);
  • useContext

在createContext时做类型约束，泛型有自动推断的功能，所以useContext就不需要再写上类型了

import type { FC } from 'react';
import React, { useContext } from 'react';

interface IProps {
  name: string;
}

interface IContext {
  prefix: (origin: string) => string;
}

const context = React.createContext<IContext>({ prefix: (str) => `prefix${str}` });

const Hello: FC<IProps> = ({ name, children }) => {
  // 根据创建context时的类型推断
  const { prefix } = useContext(context);

  return (
    <div>
      <div>hello!{prefix(name)}</div>
      {children}
    </div>
  );
};

export default Hello;

# 添加第三方依赖库

当我们需要在代码库中添加第三方依赖的时候，要做的第一件事就是查看这个库是否拥有一个带有 TypeScript 类型定义 @types 包，可以通过运行下面的命令：

# yarn
$ yarn add @types/<package-name>

# npm
$ npm install @types/<package-name>

比如我们想在项目中引入 jest，可以像下面这样安装 jest 依赖：

#yarn
$ yarn add @types/jest

#npm
$ npm install @types/jest

如果没有找到带有 TypeScript 类型定义 @types 包，例如 loadsh 在 npm 上是没有 @types/loadsh 的，该怎么办呢？ 首先通过 npm 或者 yarn 正常安装 loadsh 的依赖：

#yarn
$ yarn add loadsh

#npm
$ npm install loadsh

然后在项目中的新建 typings.d.ts 文件，为 loadsh 依赖添加 declare，加入以下代码：

declare module 'loadsh' {
  const classes: any;
  export default classes;
}

然后在 tsconfig.json 里的 include 添加上 typings.d.ts:

{
  "include": ["typings.d.ts", "src/**/*"]
}

正常引入即可：

import _ from 'loadsh';
