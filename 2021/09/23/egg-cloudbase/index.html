结合Egg，入门腾讯云Cloudbase开发

Thu Sep 23 2021
  • Node
  • Egg
  • CloudBase

不想做全栈的前端不是好前端。

# 技术栈

  • Egg (opens new window)

  • Typescript (opens new window)

  • 腾讯云开发 (opens new window)

数据库直接使用腾讯云数据库，方便快捷。

# 开始

按照cloudbase-framework (opens new window)文档：

  1. 安装 CLI
npm install -g @cloudbase/cli@latest
  1. 初始化应用
cloudbase init

在接下来的命令行提示中选择云环境（提前开通，前往腾讯云 (opens new window)）和应用模版为 Egg 应用。

  1. 新增 Typescript 支持

官方的模版是 JS 版，因此我们要手动增加 TS 支持。

npm install typescript -D

创建tsconfig.json：

{
  "compileOnSave": true,
  "compilerOptions": {
    "target": "es2017",
    "module": "commonjs",
    "strict": true,
    "noImplicitAny": false,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,
    "charset": "utf8",
    "allowJs": false,
    "pretty": true,
    "noEmitOnError": false,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "allowUnreachableCode": false,
    "allowUnusedLabels": false,
    "strictPropertyInitialization": false,
    "noFallthroughCasesInSwitch": true,
    "skipLibCheck": true,
    "skipDefaultLibCheck": true,
    "inlineSourceMap": true,
    "importHelpers": true
  },
  "exclude": ["app/public", "app/views"]
}

然后将原先的 js 文件改为 ts 文件，语法稍作修改即可。

在package.json中新增编译命令

{
  "scripts": {
    "tsc": "ets && tsc -p tsconfig.json"
  }
}
  1. 修改 Eslint 配置

因为新增了 TS 支持，则原先的 Eslint 配置要修改。

npm install @typescript-eslint/eslint-plugin @typescript-eslint/parser -D

修改.eslintrc：

{
  "root": true,
  "parser": "@typescript-eslint/parser",
  "plugins": ["@typescript-eslint"],
  "extends": ["eslint:recommended", "plugin:@typescript-eslint/recommended"]
}
  1. 启动

得益于egg-bin已经内置了ts-node，在开发期会自动编译运行，只要配置package.json即可，具体可查看文档 (opens new window)

{
  "name": "showcase",
  "egg": {
    "typescript": true,
    "declarations": true
  },
  "scripts": {
    "dev": "egg-bin dev",
    "test-local": "egg-bin test",
    "clean": "ets clean"
  }
}

然后,npm run dev 即可。
